Vibe Coding: Rapid Feature Development

Experience the speed and simplicity of Vibe mode by adding real features to the micro-blogging application

Overview

Vibe Coding is Kiro's conversational mode for rapid feature implementation. Unlike traditional AI coding assistants that require detailed instructions, Vibe mode leverages your steering documents to understand context automatically, allowing you to describe what you want in natural language and get production-ready code.

In this module, you'll experience the speed and simplicity of Vibe mode by adding real features to the micro-blogging application.

When to Use Vibe Mode

Vibe mode excels at:

  • Quick UI additions - Adding buttons, forms, or visual elements
  • Simple feature enhancements - Extending existing functionality
  • Bug fixes - Addressing specific issues in isolated components
  • Refactoring - Improving code structure in single files

Vibe mode is not ideal for:

  • Complex, multi-file features requiring architectural changes
  • Features requiring detailed requirements and design documentation
  • Changes affecting multiple system boundaries
Key Principle

Vibe mode works best when you can describe the desired outcome in a single, focused prompt. If you find yourself writing multiple paragraphs of requirements, consider using Spec mode instead.

How Vibe Mode Uses Steering

Before you write a single prompt, Kiro has already read your steering documents:

  • product.md - Understands what features exist and user personas
  • tech.md - Knows your React + TypeScript + Vite stack and patterns
  • structure.md - Understands where components live and how they're organized

This means you don't need to explain:

  • "Use TypeScript"
  • "Follow React hooks patterns"
  • "Match the existing button styling"
  • "Put it in the right directory"

Kiro already knows. You just describe what you want, not how to build it.

What You'll Build

In this module, you'll add a Cancel button to the New Post page - a feature that demonstrates Vibe mode's strengths for focused, single-component changes.

This exercise will help you understand:

  • How to write effective Vibe prompts
  • When Vibe mode is the right choice
  • How steering documents work behind the scenes
  • The iterative nature of conversational coding

The Vibe Workflow

Select Vibe Mode
Write Prompt
Kiro Generates Code
Review Diff
Looks Good?
Accept & Test

The workflow is iterative and conversational:

  1. Open Kiro Chat - Start a new session
  2. Select Vibe Mode - Choose conversational coding
  3. Write Your Prompt - Describe what you want in natural language
  4. Review the Diff - Examine what Kiro changed
  5. Test It - Verify the feature works as expected
  6. Iterate if Needed - Refine with follow-up prompts
Pro Tip

Vibe mode is iterative. Your first prompt doesn't need to be perfect. Start with the core functionality, test it, then refine with follow-up prompts like "make the button red" or "add an icon to the left of the text."

Example Prompts: Good vs. Poor

Understanding what makes a good Vibe prompt helps you get better results faster.

Example 1: Adding a UI Element

Poor prompt: "Add a cancel button to the new post page that looks nice and works properly with the existing form"

Good prompt: "Add a Cancel button next to the Submit button on the New Post page. It should clear the form and navigate back to the feed."

Example 2: Styling Changes

Poor prompt: "Make the buttons look better"

Good prompt: "Make the Cancel button a secondary style with a gray background and white text, matching the design system"

Example 3: Bug Fixes

Poor prompt: "Fix the bug"

Good prompt: "The form doesn't clear after submitting a post. Fix the reset logic in the NewPost component"

Before You Start

Before diving into the exercise, make sure you have:

  • Application running - Frontend dev server should be running at http://localhost:5173. If not: cd frontend && npm run dev
  • Steering documents generated - Check that .kiro/steering/ contains product.md, tech.md, and structure.md. If not: Go back and generate them
  • Kiro chat panel open - Look for the chat icon in the left sidebar or press Cmd/Ctrl + Shift + K
  • Test data created - You should have at least 1-2 posts in your feed. If not: Create a few posts through the UI
Quick check

Open your browser to http://localhost:5173 and verify you can see the feed with your posts. This confirms everything is working before you start making changes.
