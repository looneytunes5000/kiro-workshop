Vibe Coding: Rapid Feature Development

Experience the speed and simplicity of Vibe mode by adding real features to the paraphrasing mentoring application

Overview

Vibe Coding is TRAE SOLO's conversational mode for rapid feature implementation. Unlike traditional AI coding assistants that require detailed instructions, Vibe mode leverages your steering documents to understand context automatically, allowing you to describe what you want in natural language and get production-ready code.

In this module, you'll experience the speed and simplicity of Vibe mode by integrating a local LLM for paraphrase quality evaluation.

When to Use Vibe Mode

Vibe mode excels at:

  • Quick UI additions - Adding buttons, forms, or visual elements
  • Simple feature enhancements - Extending existing functionality
  • Bug fixes - Addressing specific issues in isolated components
  • Refactoring - Improving code structure in single files

Vibe mode is not ideal for:

  • Complex, multi-file features requiring architectural changes
  • Features requiring detailed requirements and design documentation
  • Changes affecting multiple system boundaries
Key Principle

Vibe mode works best when you can describe the desired outcome in a single, focused prompt. If you find yourself writing multiple paragraphs of requirements, consider using Spec mode instead.

How Vibe Mode Uses Steering

Before you write a single prompt, TRAE SOLO has already read your steering documents:

  • product.md - Understands what features exist and user personas
  • tech.md - Knows your React + TypeScript + Vite stack and patterns
  • structure.md - Understands where components live and how they're organized

This means you don't need to explain:

  • "Use TypeScript"
  • "Follow React hooks patterns"
  • "Match the existing button styling"
  • "Put it in the right directory"

TRAE SOLO already knows. You just describe what you want, not how to build it.

What You'll Build

In this module, you'll add an LLM evaluation panel to the paraphrase results page - a feature that demonstrates Vibe mode's strengths for focused API integration and UI changes.

This exercise will help you understand:

  • How to write effective Vibe prompts for LLM integration
  • When Vibe mode is the right choice for API-driven features
  • How steering documents work behind the scenes
  • The iterative nature of conversational coding

Looks Good?
Accept & Test

The workflow is iterative and conversational:

  1. Open TRAE SOLO Chat - Start a new session
  2. Select Vibe Mode - Choose conversational coding
  3. Write Your Prompt - Describe what you want in natural language
  4. Review the Diff - Examine what TRAE SOLO changed
  5. Test It - Verify the feature works as expected
  6. Iterate if Needed - Refine with follow-up prompts
Pro Tip

Vibe mode is iterative. Your first prompt doesn't need to be perfect. Start with the core functionality, test it, then refine with follow-up prompts like "add color coding to the scores" or "show improvement suggestions below the score."

Example Prompts: Good vs. Poor

Understanding what makes a good Vibe prompt helps you get better results faster.

Example 1: Adding LLM Integration

Poor prompt: "Add AI evaluation to the paraphrase page that looks nice"

Good prompt: "Add an evaluation panel next to the paraphrase output that calls the Ollama API at localhost:11434. Send the paraphrased text and display the quality score returned."

Example 2: UI Display Changes

Poor prompt: "Make the scores look better"

Good prompt: "Display the evaluation score with color coding: green for scores above 7, yellow for 4-7, and red for below 4. Add a progress bar visual."

Example 3: Bug Fixes

Poor prompt: "Fix the evaluation"

Good prompt: "The evaluation panel shows an error when Ollama is not running. Add a friendly message explaining that Ollama needs to be started on localhost:11434."

Before You Start

Before diving into the exercise, make sure you have:

  • Application running - Frontend dev server should be running at http://localhost:5173. If not: cd frontend && npm run dev
  • Steering documents generated - Check that .trae/steering/ contains product.md, tech.md, and structure.md. If not: Go back and generate them
  • TRAE SOLO chat panel open - Look for the chat icon in the left sidebar or press Cmd/Ctrl + Shift + K
  • Ollama running locally - Ollama should be installed and running on localhost:11434. If not: ollama serve
  • Test data created - You should have at least 1-2 paraphrased texts in your dashboard. If not: Submit some texts through the UI
Quick check

Open your browser to http://localhost:5173 and verify you can see your paraphrased texts in the dashboard. Also verify Ollama is running by visiting http://localhost:11434. This confirms everything is working before you start making changes.
