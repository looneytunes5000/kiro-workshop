Exercise: Fixing Deployment Issues

Use Spec mode to figure out why your app won't deploy — and fix it step by step

What We're Doing

In the previous page, you learned how to deploy your app. But what happens when deployment fails? Error messages, missing files, things that work on your computer but break in the cloud — this is where things get interesting.

In this exercise, you'll use TRAE SOLO's Spec mode to diagnose and fix deployment problems for the paraphrasing mentoring app. Unlike the Practice Mode exercise where you built something new from scratch, this is a "something is broken, let's fix it" scenario.

Choosing Between Vibe Mode and Spec Mode

Before we dive in, it's worth understanding when to use each mode for troubleshooting:

  • Vibe Mode is great for quick, simple fixes — like "the button text is cut off" or "add an error message when the upload fails." You describe the problem, and the AI suggests a fix. Fast and iterative.
  • Spec Mode (Spec-Driven Development) is better when the problem is deeper or you're not sure what's wrong. It forces a structured approach: understand the issue, design a fix plan, and execute systematically.
When to Spec vs. Vibe

If you can describe the fix in one sentence, use Vibe. If you're staring at an error log and aren't sure what's causing it, use Spec.

Common Deployment Problems (And Why They Happen)

Deployment scripts can fail for many reasons. Here are the most common ones, explained in plain English:

  • Wrong passwords or keys — Your app uses "environment variables" (like secret passwords) to connect to services. If these don't match between your computer and the server, things break.
  • Missing pieces — Your app needs certain tools or services installed. If the server doesn't have them, the app won't run.
  • Wrong addresses — File paths or web addresses that work on your laptop might be different on the server.
  • Permission issues — The server account running your app might not have the right to read certain files or write to certain folders.
  • Version mismatches — Your computer has newer versions of software than the server does. What works locally might break in production.
  • Connection problems — Your app might need to talk to other services (like Ollama for AI). If those services aren't reachable from the server, features fail.

Step 1: Tell TRAE What's Wrong

Open TRAE SOLO and switch to Spec mode. Describe what you're seeing:

▶ Click to reveal the deployment troubleshooting prompt
The deployment pipeline for the paraphrasing app is failing. The build succeeds locally, but the deployment to the cloud environment fails with errors. The LLM evaluation feature works locally with Ollama but fails in production. Please analyse the deployment scripts, identify the root causes, and create a plan to fix them.
Tip

Include any error messages you've seen. The more detail you give TRAE, the better its diagnosis will be.

Step 2: Read the Diagnosis

TRAE will first analyse your problem and write up a diagnosis. Think of it like a doctor's diagnosis — it should cover:

  • What's broken (each failure point)
  • Why it's broken (root cause)
  • What else breaks because of it (ripple effects)
  • Which problems to fix first (priority order)

Read through the diagnosis. Does it match what you're seeing? Did TRAE miss anything? You can ask it to refine its analysis.

Step 3: Review the Fix Strategy

Next, TRAE designs a treatment plan — a strategy for how to fix everything. Look for:

  • What files need to change
  • What settings need updating
  • How to test that the fixes work

Make sure the planned fixes won't break anything else. A good fix solves the problem without creating new ones.

Step 5: Watch TRAE Execute the Fixes

As TRAE works through each fix:

  1. Review the changes it makes to your files
  2. Check that each fix addresses the original problem
  3. Test the deployment after major changes
  4. Note any new issues that come up
