Sub-Agents - Parallel task delegation

Delegate independent tasks to parallel AI agents for faster development

What are Sub-Agents?

Sub-agents in TRAE SOLO allow you to delegate independent tasks to parallel AI agents. Instead of waiting for one task to complete before starting the next, you can run multiple tasks simultaneously, significantly speeding up your development workflow.

Key Concept

Sub-agents work independently on isolated tasks. They don't share context with each other, which prevents conflicts and ensures each agent can focus on its specific assignment.

When to Use Sub-Agents

Sub-agents are ideal for:

  • Independent File Changes: Modifying multiple files that don't depend on each other
  • Parallel Testing: Running different test suites simultaneously
  • Multi-Feature Development: Building separate features that don't interact
  • Code Refactoring: Refactoring different modules in parallel
  • Documentation Updates: Updating documentation for multiple components

How to Use Sub-Agents

  1. Identify Independent Tasks: Break your work into tasks that don't depend on each other
  2. Launch Sub-Agents: Use the Agent Panel to spawn sub-agents for each task
  3. Monitor Progress: Track each sub-agent's progress independently
  4. Review Results: Review and merge the results from each sub-agent

Example: Parallel Feature Development

Imagine you need to add three independent features to your application:

  • Sub-Agent A: Add user profile page
  • Sub-Agent B: Add notification settings
  • Sub-Agent C: Add dark mode toggle

Since these features don't interact, you can run all three sub-agents in parallel instead of sequentially.

Important

Only use sub-agents for truly independent tasks. If tasks depend on each other's output, run them sequentially to avoid conflicts and inconsistent results.

Best Practices

  • Clear Task Definitions: Give each sub-agent a clear, specific task description
  • Isolated Scope: Ensure tasks don't modify the same files or shared state
  • Monitor Resources: Be aware of resource usage when running multiple agents
  • Review Carefully: Review each sub-agent's output before merging
  • Use for Repetitive Tasks: Sub-agents excel at repetitive, well-defined tasks

Limitations

  • Sub-agents cannot share context or communicate with each other
  • Tasks that modify the same files should not run in parallel
  • Complex tasks with many dependencies are better suited for sequential execution
  • Resource usage increases with each additional sub-agent
