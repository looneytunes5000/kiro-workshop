Hook Configuration

Learn how to configure hooks for your specific project needs

Configuration File Location

Hooks are configured in the .trae/hooks.json file at your project root. This file is automatically loaded when you open a project in TRAE SOLO.

Basic Hook Structure

Each hook definition consists of three required fields:

{ "hooks": [ { "event": "post-write", "pattern": "**/*.ts", "command": "npx prettier --write {{file}}" } ] }

Event Types

  • pre-write - Before files are written to disk
  • post-write - After files are written to disk
  • pre-command - Before shell commands execute
  • post-command - After shell commands complete

Pattern Matching

The pattern field uses glob syntax to match file paths:

  • **/*.ts - All TypeScript files in any directory
  • src/**/*.py - All Python files under src/
  • **/*.{js,jsx} - All JavaScript and JSX files
  • **/* - All files (use with caution)

Advanced Configuration

Conditional Hooks

You can add conditions to control when hooks execute:

{ "hooks": [ { "event": "post-write", "pattern": "**/*.test.ts", "command": "npm test -- {{file}}", "condition": "CI !== 'true'" } ] }

Multiple Commands

Chain multiple commands using shell operators:

{ "hooks": [ { "event": "post-write", "pattern": "**/*.ts", "command": "npx eslint {{file}} --fix && npx tsc --noEmit" } ] }
Tip

Start with simple hooks and gradually add complexity. Test each hook individually before combining them into a comprehensive configuration.

Testing Your Hooks

Before deploying hooks to your team, test them thoroughly:

  1. Create a test file that matches your hook pattern
  2. Have TRAE SOLO modify the file
  3. Verify the hook executed correctly
  4. Check for any unintended side effects
Warning

Hooks that run automatically on every file write can significantly slow down your workflow if they're too heavy. Always measure hook execution time and optimize accordingly.

Common Configuration Patterns

Here are some proven hook configurations for different project types:

  • Frontend projects - Prettier + ESLint + TypeScript checking
  • Python projects - Black + isort + mypy
  • Go projects - gofmt + go vet + go test
  • Full-stack projects - Language-specific formatters + integration tests
