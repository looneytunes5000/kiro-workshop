Cloud Agent

Execute code in a managed cloud environment with unified runtime and dependency management

Overview

The Cloud Agent enables code analysis, running, and debugging in a fully managed cloud environment. It provides unified runtime and dependency management, ensuring stable and isolated remote execution without any local setup.

Instead of configuring your local machine for each project, the Cloud Agent provisions a clean environment on demand, executes your tasks, and returns results directly to your chat interface.

Benefits

  • No local setup needed - Avoids compatibility issues from local environment differences
  • Unified dependency management - All dependencies managed centrally and consistently
  • Isolated execution - Each task runs in a clean, isolated environment
  • Focus on the task - No distractions from environment configuration or device limitations

When to Use Cloud Agent

The Cloud Agent is ideal in these scenarios:

  • Complex projects with many dependencies - Let the cloud handle the setup while you focus on development
  • Team collaboration with consistent environments - Everyone works in the same configured environment
  • Working across different devices - Switch between machines without reconfiguring anything
  • Avoiding local setup time - Skip the "works on my machine" problem entirely
Key Principle

The Cloud Agent removes the friction of environment setup. You describe what you want to build, and the cloud provides a ready-to-use workspace with all dependencies pre-configured.

How It Works

Submit Your Task
Cloud Environment Provisions
Code Executes in Isolated Container
Results Returned to Chat
  1. Submit your task - Describe what you want to build or analyze in the chat interface
  2. Cloud environment provisions automatically - A clean, isolated container is created with the right runtime and dependencies
  3. Code executes in isolated container - Your task runs in a managed environment with no interference from other processes
  4. Results returned to your chat interface - Output, diffs, and artifacts appear directly in your conversation

Cloud vs Local

Understanding when to use Cloud Agent versus local execution helps you work more efficiently.

Use Cloud Agent When
  • Starting a new project from scratch
  • Working with complex dependency trees
  • Collaborating with a team that needs consistent environments
  • Switching between different machines or devices
  • You want to avoid "works on my machine" issues
Use Local Execution When
  • You already have a well-configured local environment
  • Working offline or with limited connectivity
  • Running quick, single-file edits that don't need execution
  • Debugging hardware-specific or OS-specific issues
