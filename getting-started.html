Getting Started

Start a new project in TRAE SOLO using spec-driven development

Overview

This guide walks you through the fastest path from idea to running code in TRAE SOLO. The workflow is intentionally simple: open a folder, write a /spec prompt, review the output, and iterate.

Prerequisite

You should already have a project idea in mind. TRAE SOLO will ask clarifying questions if your description is vague, but a clear goal speeds things up.

Step 1: Open a Folder

Create a new project by opening a folder on your computer.

  1. Launch TRAE SOLO (web or desktop client).
  2. Select Open Folder from the start screen.
  3. Choose or create an empty directory for your project.

Step 2: Write Your First /spec Prompt

With the folder open, start the chat and type your project idea as a /spec command. Include the clarifying phrase so TRAE SOLO confirms requirements before writing code.

▶ Click to reveal example prompt
/spec [describe your project idea], ask me for clarification on any requirements before implementing.

Example:

/spec Build a personal task manager with categories, due dates, and priority levels. Ask me for clarification on any requirements before implementing.

Step 3: Review and Refine with Vibe Coding

Once TRAE SOLO has clarified requirements, it generates code. Review the files, test the behavior, and give natural-language feedback to adjust or extend functionality.

  • Read the generated files in the sidebar.
  • Run the app (TRAE SOLO usually provides the command).
  • Ask for changes in plain English: "Make the button blue" or "Add a search bar."

Step 4: Iterate

Repeat Step 3 until the project meets your needs. Common iteration patterns:

  • Feature additions: "Add user authentication."
  • UI polish: "Use a card layout for the dashboard."
  • Bug fixes: "The form throws an error when I leave the date blank."
  • Refactoring: "Move the API calls into a separate service file."
Tips
  • Roll back to the previous version if the result is not satisfactory.
  • Use /spec again for large pivots; use chat for small tweaks.
  • Be mindful of the context usage and manage workflow with new tasks.
