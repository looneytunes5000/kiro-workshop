Generate Steering Documents

Step-by-step guide to generating and reviewing steering documents for your project

Generate steering documents

Let's create steering documents for the workshop project:

Step 1: Open the Steering panel

  1. Look at the left sidebar in TRAE SOLO
  2. Find and click on the TRAE SOLO features panel (it's the cute little ghost)
  3. Look for the "Generate Steering Docs" button
Alternative method: You can also open the command palette with Cmd + Shift + P (Mac) or Ctrl + Shift + P (Windows/Linux), then type TRAE SOLO: Generate Steering and press Enter.

Step 2: Generate the documents

  1. Click "Generate Steering Docs"
  2. TRAE SOLO will begin analysing your codebase

Step 3: Wait for analysis

TRAE SOLO will analyse your codebase in a new chat session. This typically takes 30-60 seconds. You'll see a progress indicator showing:

  • Analysing code files
  • Detecting patterns
  • Generating documentation
What's happening? TRAE SOLO is scanning your entire codebase including:
  • Source code files (.ts, .tsx, .js, .jsx)
  • Configuration files (package.json, tsconfig.json, etc.)
  • Project structure and organization
  • Existing documentation and comments

Step 4: Verify generation

Once complete, TRAE SOLO will let you know with a confirmation message in the chat session.

The easiest way to view these documents is from the Agent Steering section of the TRAE SOLO features panel.

Alternative method: You can also check the .trae/steering/ directory in your file explorer. You should see three new files:
  • product.md
  • tech.md
  • structure.md

Review generated content

Open the steering documents from the Agent Steering panel and review the three generated files.

What to look for in each file

In product.md:

  • Application name and description
  • Core features list
  • User personas or target audience
  • Architecture overview

In tech.md:

  • Frontend framework and libraries (React, TypeScript, Vite)
  • Backend runtime and services (Node.js, serverless functions, database)
  • Build commands and scripts
  • Testing frameworks

In structure.md:

  • Directory organization
  • Key file locations
  • Module responsibilities
  • Important patterns or conventions
Living documents: Steering documents should evolve with your codebase. Update them when you add new technologies, refactor architecture, or establish new patterns.

Test steering context - Before and After

Let's see the difference steering documents make.

Without steering context (hypothetical)

If you asked TRAE SOLO without steering:

Prompt: "Add a new API endpoint for user profiles"

TRAE SOLO would need to:

  • Analyse multiple files to understand your framework and architecture
  • Search through the codebase to find patterns and conventions
  • Consume significant context window space reading configuration files
  • Take longer to respond while gathering this information

With steering context (now)

Now test that TRAE SOLO understands your project context:

  1. Open a new TRAE SOLO chat session
  2. Choose Vibe mode
  3. Try this prompt:

Test prompt: "What tech stack is this project using?"

You should see:

  • At the top of the chat, TRAE SOLO displays "Including Steering Documents" with product.md, structure.md, and tech.md listed
  • TRAE SOLO responds with specific information (by referencing the steering files, not analysing code):
    • Application name and purpose (paraphrasing mentoring app)
    • Frontend stack (React 18, TypeScript, Vite)
    • Backend stack (Node.js, serverless functions, database)
    • Architecture overview
Key insight: Notice how TRAE SOLO provides specific, accurate information without analysing your codebase. It's reading directly from the steering documents (product.md and tech.md) that were generated earlier. This is much faster and more efficient than scanning through code files.

Try a more specific question

Now try something more technical:

Advanced test prompt: "Where should I add a new API endpoint and what pattern should I follow?"

TRAE SOLO should reference structure.md to tell you:

  • Create it in the appropriate backend directory
  • Follow the pattern of existing handlers
  • Use the standard SDK client libraries
  • Export handler functions properly

Reflection

Before moving on, consider:

  • How might steering documents reduce the time spent explaining context to AI? Instead of repeating your tech stack and architecture in every conversation, TRAE SOLO has persistent knowledge that's always available.
  • What information about your current projects would be valuable in steering documents? Think about the questions you repeatedly answer for new team members - those same answers belong in steering docs.
  • How does this compare to traditional documentation? Steering docs are automatically loaded and always up-to-date with your codebase, unlike wikis or README files that often become stale.
