TRAE — AI Coding Agent

An autonomous AI agent that plans, builds, tests, and deploys your entire project — right inside the TRAE workspace

What is TRAE?

TRAE is an AI coding agent that autonomously plans and executes the entire development process —from understanding your requirements, to generating code, testing, previewing results, and deploying.

You provide goals through natural language, voice, or file uploads. The agent independently breaks down tasks and works through them, greatly simplifying your development workflow.

Note: For the aiHackathon, TRAE is the recommended AI coding agent for building your projects.

Getting Started

When you open TRAE, the AI coding agent is ready to work. Use the mode toggle in the top-left corner to make sure you're in agent mode — then just start describing what you want to build.

Capabilities

Autonomous Coding

The coding agent is your AI teammate that writes code for you. You describe what you want to build, and the agent figures out how to make it happen —from planning the architecture to writing the actual code.

Instead of coding everything step by step yourself, you give the agent your goal, review the plan it generates, and let it do the heavy lifting. You stay in control: approve the plan, watch progress, and step in whenever you need to.

Task Management

TRAE supports managing multiple tasks simultaneously within a single project, breaking the limitations of traditional serial task execution. Assign parallel work streams, set priorities, and let the agent handle them concurrently to boost efficiency.

Built-in Tools

TRAE provides a range of built-in tools, including an editor, documentation viewer, and browser. Use these tools to see files, read docs, and preview your app —all without leaving the workspace.

aiHackathon Notice

Paid premium features are not permitted during the aiHackathon. This includes Figma AI, Supabase, Payment service integrations, and other subscription-based services. Participants should use free-tier services, local development, and built-in features only (Editor, DocView, Terminal, Browser, DiffView, Agents, MCP, Settings).

aiHackathon Resource

Ollama is available as a resource during the aiHackathon. If your solution requires AI models to power your application, you can use Ollama to run models locally. Ollama is included in the Survival Kit section —check there for quick access and setup guidance.

DiffView

After the agent completes its work on your project, you can click the Open Diff button in the chat panel to review all changes. The DiffView window displays:

  • The number of affected files
  • The total lines of code changed
  • A list of modified files

Click any file to view its specific changes, presented in a line-by-line diff format.
