(Optional) Deploy the Backend Yourself

Manual deployment instructions for running the workshop in your own environment

When do you need this? Follow these steps if you are running the workshop in your own environment, outside of a hosted event. If you're attending a hosted event, the backend is already deployed for you — skip this page and continue with the main setup instructions.

Overview

To complete the workshop exercises, you need a running backend. The deployment script included in the project handles everything: installing dependencies, building serverless function packages, and deploying the full stack.

This is a one-time setup step. The goal here is not to learn about deployment — it's to get the infrastructure in place so you can focus on building features with TRAE SOLO.

Prerequisites

Before running the deployment, make sure you have:

  • Node.js 18+ installed
  • A package manager (npm, yarn, or pnpm)
  • A hosting environment for serverless functions (e.g., Vercel, Netlify, Railway, or your own server)
  • A database service (e.g., MongoDB, PostgreSQL, or MySQL)
  • An authentication provider (e.g., Auth0, Firebase Auth, or Cognito)
  • The trae-solo-workshop project open in TRAE SOLO
Environment requirements: You need a hosting environment that supports serverless functions or a Node.js runtime, a database, and an authentication service. The exact services don't matter — the key is having API endpoints, data storage, and user auth.

Step 1: Configure Your Environment

Set up the required environment variables for your backend deployment.

Create a .env file

In the backend/ directory, create a .env file with your configuration:

DATABASE_URL=your-database-connection-string AUTH_PROVIDER_URL=your-auth-provider-url AUTH_CLIENT_ID=your-auth-client-id API_BASE_URL=your-api-base-url
Configuration: Replace the placeholder values with your actual service credentials. The exact variable names may vary depending on your chosen services.

Step 2: Install Dependencies and Deploy

From the root of the trae-solo-workshop project, install dependencies and deploy:

Install Dependencies

cd backend npm install

Deploy

Use your hosting platform's deployment command. Common options:

# Vercel vercel deploy --prod # Netlify netlify deploy --prod # Railway railway up # Docker (self-hosted) docker build -t workshop-backend . docker run -p 3000:3000 --env-file .env workshop-backend
Deployment time: This takes approximately 5–10 minutes depending on your hosting platform. Don't close the terminal or interrupt the process.

Step 3: Save the Outputs

When the deployment completes, note down your API endpoint and configuration values:

API_URL=https://your-app.your-hosting.com/api AUTH_USER_POOL_ID=your-user-pool-id AUTH_CLIENT_ID=your-client-id AUTH_IDENTITY_POOL_ID=your-identity-pool-id

Copy these four values — you'll need them to configure the frontend .env file in the next step of the main setup instructions.

Troubleshooting

  • "Dependencies not found" — Make sure you ran npm install in the backend directory before deploying.
  • "Environment variables missing" — Verify your .env file is properly configured and uploaded to your hosting platform.
  • "Deployment failed" — Check your hosting platform's deployment logs for details. Common causes: missing environment variables, incorrect build commands, or resource limits.
  • "API not responding" — Verify your API endpoint is correct and the server is running. Check the hosting platform's dashboard for status.
Once the deployment is complete, go back to the main Understanding the Application page and continue from Part 2: Locate Your Backend Configuration — but instead of looking at the Event Dashboard, use the configuration values from your deployment.
