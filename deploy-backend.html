(Optional) Deploy the Backend Yourself

Manual deployment instructions for running the workshop in your own AWS account

When do you need this? Follow these steps if you are running the workshop in your own AWS account, outside of an AWS-hosted event. If you're attending an AWS event, the backend is already deployed for you — skip this page and continue with the main setup instructions.

Overview

🚀 To complete the workshop exercises, you need a running backend on AWS. The deployment script included in the project handles everything: installing dependencies, building Lambda packages, bootstrapping CDK, and deploying the full stack.

This is a one-time setup step. The goal here is not to learn about deployment — it's to get the infrastructure in place so you can focus on building features with Kiro.

Prerequisites

Before running the deployment, make sure you have:

  • AWS CLI installed and configured with credentials (aws sts get-caller-identity should return your account info)
  • Node.js 22+ installed
  • AWS CDK installed globally (npm install -g aws-cdk)
  • The kiro-workshop project open in Kiro
AWS permissions required: Your AWS credentials need permissions to create Lambda functions, API Gateway, DynamoDB tables, Cognito resources, IAM roles, and CloudFormation stacks. If you're using an admin-level role or a broad development role, you should be fine.

Step 1: Set environment variables

Open a terminal in Kiro (Terminal → New Terminal) and set the required environment variables.

Bash (macOS/Linux)

export AWS_REGION="us-east-1" export STACK_NAME="KiroWorkshop"

PowerShell (Windows)

$env:AWS_REGION = "us-east-1" $env:STACK_NAME = "KiroWorkshop"
Region choice: You can use any AWS region that supports all required services (Lambda, API Gateway, DynamoDB, Cognito). us-east-1 is recommended for simplicity.

Step 2: Run the deployment script

From the root of the kiro-workshop project, run the deployment script:

Bash (macOS/Linux)

./deploy-backend.sh

PowerShell (Windows)

.\deploy-backend.ps1

The script will:

  1. Install backend dependencies
  2. Build and package Lambda functions
  3. Bootstrap AWS CDK (if not already done)
  4. Deploy the CloudFormation stack
Deployment time: This takes approximately 5–10 minutes. You'll see progress updates as CloudFormation creates resources. Don't close the terminal or interrupt the process.

Step 3: Save the outputs

When the deployment completes, you'll see output similar to this:

Outputs: KiroWorkshop.ApiUrl = https://abc123xyz.execute-api.us-east-1.amazonaws.com/prod KiroWorkshop.UserPoolId = us-east-1_ABC123XYZ KiroWorkshop.UserPoolClientId = 1a2b3c4d5e6f7g8h9i0j KiroWorkshop.IdentityPoolId = us-east-1:12345678-1234-1234-1234-123456789abc

Copy these four values — you'll need them to configure the frontend .env file in the next step of the main setup instructions.

Troubleshooting

  • "Unable to resolve AWS account" — Your AWS credentials are not configured or have expired. Verify with aws sts get-caller-identity. If this fails, reconfigure your credentials using aws configure or by exporting temporary session credentials.
  • "Stack already exists" — A stack with the same name already exists. Either use a different name (export STACK_NAME="KiroWorkshop-2") or delete the existing stack first (cd infrastructure && cdk destroy).
  • CDK bootstrap errors — If CDK has never been bootstrapped in this account/region, the script should handle it automatically. If it fails, run manually: cdk bootstrap aws://ACCOUNT_ID/REGION (replace with your values).
  • Deployment fails midway — Check the CloudFormation events for details: aws cloudformation describe-stack-events --stack-name KiroWorkshop --max-items 10. Common causes: insufficient permissions, service limits, or resource naming conflicts.
Once the deployment is complete, go back to the main Understanding the Application page and continue from Part 2: Locate Your Backend Configuration — but instead of looking at the Event Dashboard, use the stack outputs printed in your terminal.
