Understanding the Application

Explore the application architecture and configure the frontend to connect to your deployed backend

Overview

Before you can start building features with Kiro, you need to understand the application you'll be working with. The backend infrastructure has already been deployed automatically to your AWS account. In this section, you'll explore the application architecture and configure the frontend to connect to your deployed backend.

Why work with an existing application?

Working with an existing, running application is crucial for understanding how AI can assist with real-world development scenarios. While AI can certainly help with brand-new projects, this workshop focuses on the more common scenario: enhancing and maintaining existing codebases. This approach demonstrates:

  • How Kiro understands existing code patterns and architecture
  • How AI-assisted development integrates with your current workflow
  • How to safely modify production-like applications with AI guidance
  • The importance of testing changes against a working system

Understanding the Application

The micro-blogging application you'll be working with consists of:

Backend (AWS - Already Deployed):

  • Lambda functions for API endpoints
  • DynamoDB tables for data storage
  • API Gateway for HTTP routing
  • Cognito for user authentication

Frontend (Local):

  • React + TypeScript + Vite application
  • Runs locally on your machine during the workshop
  • Connects to the deployed AWS backend

Architecture:

Users/Browsers → React Frontend (Local Dev Server)

↓ HTTPS / API Calls

API Gateway → Lambda Functions (Node.js 22)

↓ Invokes / Read/Write

DynamoDB (NoSQL Database)

↓ Authenticate / User Sessions

AWS Cognito (Authentication)

Backend Already Deployed: The backend infrastructure was automatically deployed to your AWS account when you started this workshop. You don't need to run any deployment commands - the infrastructure is ready to use!

Part 1: Access the Source Code

To participate in this workshop, you need to get a copy of the application source code.

Step 1: Download the Code

Click the download link in the workshop to obtain a zipped version of the source.

Verify download: Check your Downloads folder for kiro-workshop-assets.zip. The file should be approximately 140 KB.

Step 2: Unzip the Application

Unzip the source in the folder of your choice. After extraction, you should see the following structure:

kiro-workshop/ ├── frontend/ # React application │ ├── src/ │ │ ├── components/ # UI components │ │ ├── pages/ # Page components │ │ ├── services/ # API service layer │ │ └── contexts/ # React contexts │ ├── package.json │ └── .env.example # Environment template ├── backend/ # Lambda functions (for reference) │ ├── src/ │ │ └── functions/ # API handlers │ └── package.json ├── infrastructure/ # AWS CDK code (for reference) │ └── lib/ # Stack definitions └── DESIGN_LANGUAGE.md # Application design patterns

Open in Kiro

In Kiro, go to File → Open Folder and select the kiro-workshop directory you just extracted. This will be your workspace for the entire workshop.

Part 2: Locate Your Backend Configuration

Your backend infrastructure has already been deployed. You need to find the configuration values to connect your frontend to it.

Step 1: Find the Stack Outputs

  1. Navigate to the Event Dashboard
  2. Look for the Stack Outputs section
  3. You should see four values displayed (in this order):
    • ViteApiUrl - Your API Gateway endpoint
    • ViteUserPoolId - Your Cognito User Pool ID
    • ViteUserPoolClientId - Your Cognito App Client ID
    • ViteIdentityPoolId - Your Cognito Identity Pool ID
Can't find the outputs? If you don't see the Stack Outputs section on the Event Dashboard, the backend deployment may still be in progress. Wait a few minutes and refresh the page. The deployment typically takes 5-10 minutes to complete.

Step 2: Copy the Values

Keep these values handy - you'll need them in the next step to configure your frontend application.

Part 3: Configure the Frontend

Now you'll connect the frontend React application to your deployed backend.

Step 1: Update Environment File

The frontend needs to know where your backend API is located and how to authenticate users. The project includes a .env file with placeholder values that you need to update.

  1. In Kiro, navigate to the frontend/ folder in the file explorer
  2. Open the .env file
  3. You'll see the following keys already defined:
    VITE_API_URL= VITE_USER_POOL_ID= VITE_USER_POOL_CLIENT_ID= VITE_IDENTITY_POOL_ID=
  4. Copy each value from the Stack Outputs on the Event Dashboard and paste it after the corresponding key
  5. Save the file (Cmd + S on Mac, Ctrl + S on Windows/Linux)
Example: Your .env file should look something like this after adding the values:
VITE_API_URL=https://abc123xyz.execute-api.us-east-1.amazonaws.com/prod VITE_USER_POOL_ID=us-east-1_ABC123XYZ VITE_USER_POOL_CLIENT_ID=1a2b3c4d5e6f7g8h9i0j VITE_IDENTITY_POOL_ID=us-east-1:12345678-1234-1234-1234-123456789abc

Step 2: Install Dependencies and Start Dev Server

With the configuration in place, open a terminal in Kiro (Terminal → New Terminal) and install the frontend dependencies:

cd frontend npm install npm run dev

You should see:

VITE v5.0.0 ready in 500 ms ➜ Local: http://localhost:5173/
Keep This Running: Leave this terminal running throughout the workshop. The dev server will automatically reload when you make code changes with Kiro.

Part 4: Test the Application

Your isolated environment: The entire backend infrastructure is deployed in your AWS account. When you register, you're creating a user in your own AWS Cognito user pool—your data stays in your account.

Step 1: Open and Register

  1. Open a web browser and navigate to http://localhost:5173
  2. Click "Register"
  3. Create a new account (this creates a user in your AWS Cognito user pool)
Password Requirements: The registration form doesn't display password requirements, but AWS Cognito enforces them. Your password must be at least 8 characters and include:
  • At least one uppercase letter
  • At least one lowercase letter
  • At least one number
  • At least one special character (e.g., !@#$%^&*)

If registration fails with a generic error, double-check your password meets these requirements. We'll address this UX gap later in the workshop!

Once the account is created, you will be redirected to the landing page.

Step 2: Create Test Data

  1. Click "New Post" in the navigation
  2. Create 2-3 test posts with different content
  3. Each time a post is created, you are redirected to the feed
Why Create Test Data? Having posts in your feed makes it easier to test the features you'll build with Kiro. You'll be able to see your changes immediately.

Validation Checklist

Before proceeding, verify:

  • Source code downloaded and opened in Kiro
  • Stack Outputs visible on workshop welcome page
  • .env file created with correct values
  • Frontend dev server is running
  • Application loads in browser at http://localhost:5173
  • You can register and log in
  • You can create posts and see them in the feed

Troubleshooting

Frontend Configuration Issues:

  • "Network Error" when logging in - Check VITE_API_URL in .env matches the ApiUrl from Stack Outputs
  • "User pool does not exist" - Verify VITE_USER_POOL_ID is correct and matches Stack Outputs
  • Missing Stack Outputs - The backend deployment may still be in progress. Wait a few minutes and refresh the workshop page
  • Port 5173 already in use - Stop other Vite servers

Frontend Dependency Issues:

Dependencies fail to install - Clear cache and retry:

cd frontend rm -rf node_modules package-lock.json npm install

Windows (PowerShell):

cd frontend Remove-Item -Recurse -Force node_modules, package-lock.json -ErrorAction SilentlyContinue npm install

Backend Status Check:

To verify your backend deployment status:

aws cloudformation describe-stacks --stack-name CodeServer --query 'Stacks[0].StackStatus'

Expected status: "CREATE_COMPLETE" or "UPDATE_COMPLETE"

Reflection

Before moving on, consider:

  • Why do you think we work with a deployed application? Having a working application allows you to test changes immediately and understand how new features integrate with existing functionality.
  • How does having a working application help with AI-assisted development? AI can better understand context when it can analyze a complete, running system. It can see how components interact, follow existing patterns, and suggest changes that fit the current architecture.
  • Why focus on existing projects? AI is powerful for both new and existing projects. This workshop focuses on existing codebases because that's where most developers spend their time in real-world scenarios. Learning to work with AI on existing code gives you skills that transfer directly to your daily work—understanding how AI learns from established patterns, maintains consistency, and integrates changes safely into production systems.
