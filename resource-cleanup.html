Resource Cleanup

Clean up any resources you created during the workshop

Why Clean Up?

It's important to clean up resources after completing the workshop to avoid ongoing charges and to keep your development environment organised. Good cleanup habits are essential for any cloud or local development workflow.

Important

If you deployed any cloud resources during this workshop, make sure to clean them up to avoid ongoing charges. Check your cloud provider's console for active resources.

Local Resources

Don't forget to clean up local resources as well:

  • Remove local test data: Delete any temporary files or test databases created during exercises
  • Close development servers: Stop any running local servers or background processes
  • Clean up test credentials: Remove any temporary configuration files or test credentials

Cloud Resources

If you deployed any cloud services during the workshop:

  • Check cloud provider console: Log in to your cloud provider's dashboard and review active resources
  • Delete any deployed services: Remove any applications, functions, or services you deployed
  • Remove test databases: Delete any database instances or tables created for testing
Note

Some resources may take a few minutes to fully delete. If you encounter errors, wait a few minutes and try again.

Verify Cleanup

After cleanup, verify that resources have been removed:

  • Check your cloud provider's console for any remaining resources
  • Verify that local development servers are stopped
  • Confirm test databases and temporary files are deleted
