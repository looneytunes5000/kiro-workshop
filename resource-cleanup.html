Resource Cleanup

Clean up AWS resources created during the workshop

Why Clean Up?

It's important to clean up resources after completing the workshop to avoid ongoing charges and to keep your AWS account organized. Even if resources are covered by the AWS Free Tier, good cleanup habits are essential for cloud development.

Important

Some resources may incur charges if left running. Complete this cleanup section before ending your workshop session.

Resources to Clean Up

Resource Type Service Cleanup Method
Backend API AWS Lambda / API Gateway Delete stack or service
Database Amazon DynamoDB Delete table
Storage Amazon S3 Empty and delete bucket
Authentication Amazon Cognito Delete user pool

Cleanup Steps

  1. Navigate to AWS Console: Open the AWS Management Console in your browser
  2. Go to CloudFormation: Navigate to the CloudFormation service
  3. Select Workshop Stack: Find the stack created for this workshop
  4. Delete Stack: Select the stack and click "Delete"
  5. Verify Deletion: Wait for the stack deletion to complete and verify all resources are removed

Alternative: AWS CLI Cleanup

If you prefer using the command line:

# Delete the CloudFormation stack aws cloudformation delete-stack --stack-name kiro-workshop-stack # Wait for deletion to complete aws cloudformation wait stack-delete-complete --stack-name kiro-workshop-stack

Verify Cleanup

After cleanup, verify that resources have been removed:

  • Check CloudFormation for deleted stacks
  • Verify Lambda functions are removed
  • Confirm DynamoDB tables are deleted
  • Check S3 buckets are empty and removed
Note

Some resources may take a few minutes to fully delete. If you encounter errors, wait a few minutes and try again.

Local Cleanup

Don't forget to clean up local resources as well:

  • Remove any temporary files created during exercises
  • Clean up local database instances if you created any
  • Remove test credentials or configuration files
