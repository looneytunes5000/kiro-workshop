Add LLM Evaluation

Use Vibe mode to integrate a local LLM (Ollama) for paraphrase quality evaluation

Exercise Overview

In this exercise, you'll use TRAE SOLO's Vibe mode to add an LLM evaluation panel to the paraphrase results page. This exercise demonstrates how Vibe mode can quickly integrate external APIs and display the results in your UI.

Why Vibe Mode?

Adding LLM evaluation is a focused task that Vibe mode handles well:

  • It involves a single API call to Ollama
  • The UI component is self-contained
  • Steering documents provide the React/TypeScript context
  • You can iterate quickly with follow-up prompts
Prerequisite

Make sure Ollama is installed and running on your machine. You can start it with ollama serve. The default API endpoint is http://localhost:11434.

Step 1: Open TRAE SOLO Chat

Open the TRAE SOLO chat panel and make sure Vibe mode is selected. You should see the conversational interface ready for your prompt.

Step 2: Write Your Prompt

Describe what you want clearly and specifically:

Add an evaluation panel to the paraphrase results page that: 1. Calls the Ollama API at http://localhost:11434/api/generate 2. Sends the paraphrased text with a prompt to evaluate quality (1-10 scale) 3. Displays the score with color coding (green for 7+, yellow for 4-6, red for 1-3) 4. Shows improvement suggestions returned by the LLM

Step 3: Review the Changes

TRAE SOLO will generate the code changes. Review the diff carefully to ensure:

  • The API call targets the correct Ollama endpoint
  • Error handling is in place for when Ollama is not running
  • The evaluation panel displays next to the paraphrase output
  • Color coding matches the specified score ranges
  • Loading states are shown while waiting for the LLM response

Step 4: Test the Feature

Accept the changes and test them:

  1. Ensure Ollama is running (ollama serve)
  2. Submit a text for paraphrasing
  3. Verify the evaluation panel appears with a quality score
  4. Check that color coding matches the score
  5. Stop Ollama and verify the error message displays correctly

Step 5: Iterate and Refine

Use follow-up prompts to improve the feature:

"Add a loading spinner while waiting for the LLM response"
"Make the evaluation panel collapsible with a toggle button"
"Add a history of previous evaluations for this text"

Key Takeaways

  • Vibe mode excels at focused API integration tasks
  • Specific prompts with endpoint details get better results
  • Always include error handling for external service calls
  • Iterate with follow-up prompts to refine the UI
  • Test with the external service both running and stopped
