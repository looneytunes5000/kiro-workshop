Going Further: Context Engineering

Advanced techniques for optimising AI context and building custom capabilities

What is Context Engineering?

Context engineering is the practice of designing and optimising the information provided to AI systems to improve their output quality. In TRAE SOLO, this means carefully curating what the AI knows about your codebase, conventions, and requirements.

Key Concept

Better context leads to better AI output. Context engineering is about providing the right information at the right time, in the right format, to maximize TRAE SOLO's effectiveness.

Context Engineering Principles

  • Relevance: Include only information that's directly relevant to the task
  • Specificity: Be specific rather than vague in your guidance
  • Structure: Organise context in a way that's easy for the AI to parse
  • Recency: Keep context up-to-date with your current codebase state
  • Progressive Disclosure: Provide context gradually as needed, not all at once

Exercise 1: Build a Private Power

Building Custom Powers

Powers in TRAE SOLO are custom capabilities you define to extend the AI's abilities. A "private power" is a power specific to your project or team.

Objective: Create a custom power that automates a repetitive task specific to your workflow.

Steps:

  1. Identify the Task: Choose a repetitive task you do frequently (e.g., generating API clients, creating component templates)
  2. Define the Power: Create a power definition file that describes the task, inputs, and expected outputs
  3. Implement the Logic: Write the logic that TRAE SOLO will execute when the power is invoked
  4. Test the Power: Invoke the power and verify it produces the expected output
  5. Refine: Adjust the power based on test results and edge cases
Tip

Start with a simple, well-defined task. Complex powers are harder to debug and maintain.

Exercise 2: Build an Agent Skill

Creating Reusable Skills

Agent skills are reusable capabilities that you can invoke across different projects. Unlike powers which are project-specific, skills are designed to be portable.

Objective: Build a skill that can be reused across multiple projects to solve a common problem.

Steps:

  1. Identify the Pattern: Find a pattern or practice that applies across multiple projects
  2. Design the Skill: Define the skill's interface, inputs, outputs, and behaviour
  3. Implement the Skill: Write the skill implementation using TRAE SOLO's skill framework
  4. Package the Skill: Package the skill for distribution and reuse
  5. Test Across Projects: Verify the skill works in different project contexts

Example Skill Ideas:

  • Database migration generator
  • API endpoint tester
  • Code style enforcer
  • Documentation generator

Exercise 3: Build a Custom Agent

Creating Specialized Agents

Custom agents are specialised AI configurations designed for specific types of work. They combine context, tools, and behaviour patterns into a reusable agent definition.

Objective: Build a custom agent optimized for a specific type of development task.

Steps:

  1. Define the Agent's Purpose: What specific type of work will this agent handle?
  2. Configure Context: Set up the context files and steering documents the agent needs
  3. Define Tools: Specify which tools the agent should have access to
  4. Set Behaviour Patterns: Define how the agent should approach problems and make decisions
  5. Test and Iterate: Run the agent on sample tasks and refine its configuration

Example Agent Types:

  • Test Agent: Specialized in writing and running tests
  • Refactor Agent: Focused on code refactoring and cleanup
  • Documentation Agent: Dedicated to writing and updating documentation
  • Security Agent: Specialized in security review and vulnerability detection

Context Engineering Best Practices

  • Start Small: Begin with minimal context and add more as needed
  • Use Examples: Include concrete examples in your context documents
  • Keep It Current: Regularly update context to match your codebase
  • Measure Results: Track how context changes affect AI output quality
  • Share Knowledge: Document effective context patterns for your team
