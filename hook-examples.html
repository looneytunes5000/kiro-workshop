Hook Examples

Real-world hook patterns for common development workflows

TypeScript/React Project

A comprehensive hook setup for a modern TypeScript React application:

{ "hooks": [ { "event": "post-write", "pattern": "**/*.{ts,tsx}", "command": "npx prettier --write {{file}} && npx eslint {{file}} --fix" }, { "event": "post-write", "pattern": "**/*.{ts,tsx}", "command": "npx tsc --noEmit" }, { "event": "post-write", "pattern": "**/*.test.{ts,tsx}", "command": "npm test -- --testPathPattern={{file}}" } ] }

Python/FastAPI Project

Hooks for a Python backend service using FastAPI:

{ "hooks": [ { "event": "post-write", "pattern": "**/*.py", "command": "black {{file}} && isort {{file}}" }, { "event": "post-write", "pattern": "**/*.py", "command": "mypy {{file}}" }, { "event": "post-write", "pattern": "tests/**/*.py", "command": "pytest {{file}} -v" } ] }

Go Microservice

Hooks for a Go-based microservice:

{ "hooks": [ { "event": "post-write", "pattern": "**/*.go", "command": "gofmt -w {{file}}" }, { "event": "post-write", "pattern": "**/*.go", "command": "go vet ./..." }, { "event": "post-write", "pattern": "**/*_test.go", "command": "go test -v ./..." } ] }

Documentation Hooks

Automatically update documentation when code changes:

{ "hooks": [ { "event": "post-write", "pattern": "**/api/**/*.ts", "command": "npx typedoc --out docs/api {{file}}" }, { "event": "post-write", "pattern": "**/*.py", "command": "pdoc --html --output-dir docs {{file}}" } ] }
Tip

Use documentation hooks to keep your API docs, type definitions, and code examples in sync with your implementation. This eliminates the common problem of outdated documentation.

Security Scanning Hooks

Run security checks on every code change:

{ "hooks": [ { "event": "post-write", "pattern": "**/*.{js,ts,py,go}", "command": "npx audit-ci --moderate" }, { "event": "post-write", "pattern": "**/*.py", "command": "bandit -r {{file}}" }, { "event": "pre-write", "pattern": "**/*", "command": "check-secrets.sh {{file}}" } ] }
Warning

Security scanning hooks can be slow. Consider running them only on specific file types or using incremental scanning to avoid blocking your workflow.

Deployment Hooks

Automate deployment tasks after code changes:

{ "hooks": [ { "event": "post-write", "pattern": "Dockerfile", "command": "docker build -t myapp:latest ." }, { "event": "post-write", "pattern": "docker-compose.yml", "command": "docker-compose up -d" }, { "event": "post-command", "pattern": "**/*", "command": "notify-deploy.sh 'Code updated by TRAE SOLO'" } ] }

Key Takeaways

  • Start with formatting hooks and gradually add more complex automation
  • Use pattern matching to target specific file types and directories
  • Chain related commands to reduce hook execution overhead
  • Test hooks thoroughly before adding them to shared configurations
  • Monitor hook performance and optimize slow-running hooks
