Advanced Steering - Domain-specific guidance

Create specialized steering documents for different domains and contexts

Beyond Basic Steering

While basic steering documents provide general guidance for your codebase, advanced steering allows you to create domain-specific guidance that adapts to different contexts, file types, and development scenarios.

Key Concept

Advanced steering uses conditional rules and domain-specific documents to provide targeted guidance based on the current context, file type, or task being performed.

Domain-Specific Steering Patterns

File-Type Based

Different steering rules for TypeScript, CSS, configuration files, and test files

Layer-Based

Separate guidance for frontend, backend, database, and infrastructure code

Task-Based

Context-specific rules for feature development, bug fixes, refactoring, and testing

Team-Based

Team-specific conventions for multi-team projects with different coding standards

Creating Advanced Steering Documents

Advanced steering documents use a more sophisticated structure:

  1. Define Context Rules: Specify when each steering document applies
  2. Create Domain Documents: Write specialized guidance for each context
  3. Set Priority Levels: Define which rules take precedence when multiple apply
  4. Test and Refine: Verify steering behavior across different scenarios

Example: Layer-Based Steering

// .trae/steering/backend.md # Backend Steering When working on backend code: - Use repository pattern for data access - All API endpoints must have input validation - Error responses follow RFC 7807 Problem Details - Use dependency injection for all services - Write integration tests for all endpoints
// .trae/steering/frontend.md # Frontend Steering When working on frontend code: - Use functional components with hooks - All components must be accessible (WCAG 2.1 AA) - State management through React Query or Zustand - CSS modules for component styling - Write unit tests with React Testing Library

Conditional Steering Rules

You can create rules that only apply in specific situations:

  • File Pattern Matching: Apply rules only to files matching specific patterns
  • Directory-Based Rules: Different guidance for different directories
  • Task Detection: Automatically detect if you're adding features, fixing bugs, or refactoring
  • Time-Based Rules: Different rules for development vs. production environments
Best Practice

Keep steering documents focused and concise. Overly complex steering can confuse TRAE SOLO and reduce effectiveness. Start with a few key rules and expand based on actual needs.

Testing Steering Effectiveness

To verify your steering documents are working:

  1. Ask TRAE SOLO to generate code in different contexts
  2. Review whether the output follows your steering rules
  3. Check if the correct steering document is being applied
  4. Refine rules that aren't being followed correctly

Steering Maintenance

As your project evolves, your steering documents should too:

  • Regular Reviews: Periodically review steering documents for accuracy
  • Update with Changes: Update steering when conventions change
  • Remove Outdated Rules: Delete rules that no longer apply
  • Add New Patterns: Document new patterns as they emerge
