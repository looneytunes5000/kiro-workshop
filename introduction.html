Introduction

Welcome to the Kiro Hands-on Workshop - explore AI-native development through practical exercises

Overview

Welcome to the Kiro Hands-on Workshop! In this workshop, you'll explore Kiro's AI-native development capabilities through practical, hands-on exercises with a real serverless application.

Kiro is an AI-native development environment that transforms how you build software. Rather than simply adding AI features to a traditional IDE, Kiro reimagines the development experience around AI collaboration. You'll learn to work with Kiro's unique capabilities to implement features faster while maintaining code quality and architectural consistency.

Prerequisites

Knowledge and experience

  • Familiarity with software development concepts
  • Basic understanding of web development (HTML, CSS, JavaScript)
  • Some experience with cloud services is helpful but not required

Required tools

  • Kiro IDE - You'll install and configure Kiro during the workshop
  • AWS Builder ID - For authentication and accessing workshop resources
  • Modern web browser - Chrome, Firefox, Safari, or Edge
  • Text editor - For any manual code edits (though Kiro handles most of this)

Demo Application

Throughout this workshop, you'll work with a serverless micro-blogging platform. This application provides a realistic codebase to practice Kiro's capabilities on, including:

  • Frontend - A web interface for creating and viewing posts
  • Backend APIs - Serverless functions handling CRUD operations
  • Database - Persistent storage for posts and user data
  • Authentication - User identity and access management

The application is designed to be simple enough to understand quickly, yet complex enough to demonstrate real-world development scenarios.

Deploying the Application

The workshop provides a pre-configured application that you'll enhance and modify using Kiro. You'll learn to:

  • Set up the development environment
  • Understand the existing codebase structure
  • Use Kiro to implement new features
  • Test and validate your changes

Kiro's Core Capabilities

Throughout this workshop, you'll explore five key capabilities that make Kiro unique:

Vibe Coding

Rapid feature development through natural language descriptions. Describe what you want in plain English, and Kiro helps you build it quickly while maintaining code quality.

Spec-Driven Development

For complex, multi-file features, use structured specifications to guide Kiro through planning and implementation. This ensures thorough consideration of requirements, edge cases, and architectural impact before writing code.

Steering

Teach Kiro about your codebase conventions, architectural patterns, and team standards. Steering documents help Kiro understand your project's unique context and generate code that fits seamlessly.

Agent Hooks

Automate repetitive workflows by configuring Kiro to respond to specific events or patterns. Agent Hooks let you customize Kiro's behavior to match your team's development practices.

MCP Integration

Extend Kiro's capabilities by connecting to external tools and services through the Model Context Protocol (MCP). This allows Kiro to interact with databases, APIs, version control, and more.

Pacing Yourself

This workshop is designed to be completed in 3-4 hours. However, you can adjust the pace based on your experience level and interests:

  • Beginners - Take your time with each exercise, read all the explanations, and don't hesitate to experiment
  • Experienced developers - You may move through familiar concepts more quickly and focus on advanced topics
  • Time-constrained - Focus on the core exercises (Steering, Vibe Coding, Spec-Driven Development) and skip optional sections

Workshop Conventions

Throughout this workshop, you'll encounter:

  • Code blocks - Shown in monospace font with syntax highlighting
  • Step-by-step instructions - Numbered lists for sequential actions
  • Tips and notes - Additional context and best practices
  • Challenge exercises - Optional tasks to deepen your understanding

Reflection Questions

As you progress through the workshop, consider these questions:

  • How does Kiro's approach differ from traditional AI code completion tools?
  • Which Kiro capability would have the biggest impact on your current workflow?
  • How could you introduce these practices to your team?
  • What types of projects would benefit most from Kiro's capabilities?
