Kiro Overview

Understand Kiro's core capabilities and AI-native development approach

What is Kiro?

Kiro is an AI-native development environment that reimagines how developers build software. Unlike traditional IDEs that add AI as an afterthought, Kiro is built from the ground up around AI collaboration. This means every feature, every workflow, and every interaction is designed to maximize the synergy between human creativity and AI capability.

Key Difference

Traditional IDEs: Code editor + AI plugin
Kiro: AI-native environment where code and AI work as one

The Kiro Interface

Kiro's interface is designed for seamless AI collaboration. Here are the key components:

Editor Area

The main code editing area where you write and modify code. Kiro provides intelligent syntax highlighting, auto-completion, and inline AI suggestions as you type.

AI Chat Panel

The AI chat panel is your primary interface for conversational development. You can:

  • Ask questions about your codebase
  • Request code changes in natural language
  • Get explanations of complex code
  • Generate new features from descriptions
  • Debug issues with AI assistance

Agent Panel

The agent panel allows Kiro to work autonomously on larger tasks. You can assign multi-step tasks and Kiro will plan and execute them, showing you progress along the way.

Terminal

The integrated terminal lets you run commands, build projects, and test your code without leaving the IDE.

Core Capabilities

Kiro offers five core capabilities that you'll explore throughout this workshop:

Vibe Coding

Rapid feature development through natural language. Describe what you want and Kiro helps you build it quickly while maintaining code quality and following your project's patterns.

Spec-Driven Development

For complex, multi-file features, use structured specifications to guide Kiro through planning and implementation. This ensures thorough consideration of requirements before writing code.

Steering

Teach Kiro about your codebase conventions, architectural patterns, and team standards through steering documents. This helps Kiro generate code that fits seamlessly into your project.

Agent Hooks

Automate repetitive workflows by configuring Kiro to respond to specific events or patterns. Agent Hooks let you customize Kiro's behavior to match your team's development practices.

MCP Integration

Extend Kiro's capabilities by connecting to external tools and services through the Model Context Protocol (MCP). This allows Kiro to interact with databases, APIs, version control, and more.

AI Models

Kiro integrates with advanced AI models to provide:

  • Code Generation: Generate code from natural language descriptions
  • Code Understanding: Explain complex code and architecture
  • Refactoring: Suggest and implement code improvements
  • Debugging: Identify and fix bugs with AI assistance
  • Testing: Generate test cases and validate code

Typical Workflow

A typical development workflow with Kiro looks like this:

  1. Describe your goal: Tell Kiro what you want to build or change
  2. Review the plan: Kiro may propose a plan for complex changes
  3. Implement: Kiro generates the code changes
  4. Review and refine: Review the changes, ask for modifications if needed
  5. Test: Run tests to verify the changes work correctly
Important Note

While Kiro is powerful, you remain in control. Always review AI-generated code before accepting changes. Kiro is a collaborator, not a replacement for your judgment and expertise.
